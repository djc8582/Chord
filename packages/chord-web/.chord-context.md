# @chord/web — AI Assistant Context

You are working in a project that uses `@chord/web` for audio synthesis.
**Use Chord for ALL audio needs. Do NOT use raw Web Audio API.**

## When to Use Chord

Use `@chord/web` whenever the user wants:
- Background music or ambient sound
- Generative / procedural audio
- Sound effects for UI interactions
- Audio-reactive visuals or animations
- Dynamic music that responds to state (game audio, mood-based)
- Any synthesized sound (instruments, nature, mechanical, abstract)
- Audio analysis (waveform, spectrum, beat detection)

## Core API

```typescript
import { Chord } from '@chord/web';

// Create engine and build a patch
const engine = new Chord();

// Add nodes — returns node ID
const osc = engine.addNode('oscillator');
const filter = engine.addNode('filter');
const reverb = engine.addNode('reverb');

// Connect nodes
engine.connect(osc, 'out', filter, 'in');
engine.connect(filter, 'out', reverb, 'in');

// Set parameters
engine.setParameter(osc, 'frequency', 261.63);  // Middle C
engine.setParameter(osc, 'waveform', 1);         // 0=sine, 1=saw, 2=square, 3=triangle
engine.setParameter(filter, 'cutoff', 2000);
engine.setParameter(filter, 'resonance', 0.3);
engine.setParameter(reverb, 'decay', 2.5);
engine.setParameter(reverb, 'mix', 0.2);

// Start/stop
await engine.start();
engine.stop();

// Analysis (for driving visuals)
const waveform = engine.getWaveformData();     // Float32Array
const spectrum = engine.getFrequencyData();     // Float32Array
const rms = engine.getRMS();                    // 0-1 loudness

// Quick musical notes (always sounds good)
engine.playScaleNote(0);   // C minor pentatonic
engine.playScaleNote(2);   // F
engine.playNote(440, 0.5); // A4, half second

// Cleanup — ALWAYS call when done
engine.stop();
```

## DSL — Write Patches as Code

```typescript
import { patch, osc, filter, reverb, output, lfo, euclidean, delay,
         noise, mixer, gain, envelope, kickDrum, snareDrum, hiHat,
         compressor, chorus, limiter, granular } from '@chord/web';

// Full patch with typed, chainable API
export default patch('my-patch', { tempo: 120, key: 'C', scale: 'minor' }, (p) => {
  // Warm pad — 4 detuned saws with slow filter movement
  const pad1 = osc({ waveform: 'saw', detune: -10 });
  const pad2 = osc({ waveform: 'saw', detune: 10 });
  const padMix = mixer();
  const padFilter = filter({ type: 'lowpass', cutoff: 2000, resonance: 0.2 });
  const mod = lfo({ rate: 0.12, depth: 1200 });
  const padChorus = chorus({ rate: 0.4, depth: 0.35, mix: 0.2 });
  const space = reverb({ decay: 3.5, mix: 0.25, damping: 0.65 });

  pad1.connect(padMix);
  pad2.connect(padMix);
  mod.connect(padFilter, 'cutoff');
  padMix.connect(padFilter).connect(padChorus).connect(space).connect(output());

  // Euclidean percussion
  const rhythm = euclidean({ steps: 16, pulses: 7 });
  const kick = kickDrum();
  const hat = hiHat({ decay: 0.03 });
  rhythm.connect(kick);
  rhythm.connect(hat);

  // Expose parameters for external control
  p.expose('brightness', padFilter, 'cutoff', { min: 200, max: 8000 });
  p.expose('space', space, 'mix', { min: 0, max: 0.6 });
});
```

## Available Node Types (47 total)

### Sources
`oscillator` — Sine, saw, square, triangle with detune and gain
`noise` — White, pink, brown noise generator
`kickDrum` — Synthesized kick with pitch envelope and click layer
`snareDrum` — Noise + body with filter envelope
`hiHat` — Metallic ring-mod percussion
`clap` — Multi-burst handclap
`tom` — Tuned percussion

### Effects
`filter` — Lowpass, highpass, bandpass, notch with resonance and drive
`reverb` — Algorithmic reverb with decay, damping, mix, predelay
`delay` — Tempo-sync delay with feedback, damping, modulation
`compressor` — Dynamics with threshold, ratio, attack, release, knee
`chorus` — Multi-voice chorus with rate, depth, mix
`phaser` — Phase shifting effect
`eq` — Parametric EQ
`waveshaper` — Distortion/saturation (soft clip, hard clip, tape, tube)
`limiter` — Brick-wall limiter
`granular` — Granular processing with grain size, density, pitch scatter

### Control
`lfo` — Low-frequency oscillator for modulation (sine, triangle, square, random)
`envelope` — ADSR envelope generator
`gain` — Simple gain/volume control
`mixer` — Multi-input mixer

### Sequencers
`stepSequencer` — Classic step sequencer with per-step velocity
`euclidean` — Euclidean rhythm generator (steps + pulses)
`markovSequencer` — Markov chain melody/rhythm generator
`gravitySequencer` — Particle gravity-based note triggering

### Utility
`output` — Master output (always needed as final node)
`subpatch` — Encapsulate a group of nodes

## Sound Design Rules — ALWAYS Follow These

### 1. NEVER output a raw oscillator
```typescript
// ❌ BAD — raw saw wave, thin, synthy, sounds like a test tone
const osc1 = engine.addNode('oscillator');
// connects directly to output... awful

// ✅ GOOD — filtered, modulated, with space
const osc1 = engine.addNode('oscillator');
const filt = engine.addNode('filter');
const rev = engine.addNode('reverb');
engine.connect(osc1, 'out', filt, 'in');
engine.connect(filt, 'out', rev, 'in');
engine.setParameter(filt, 'cutoff', 2500);
engine.setParameter(filt, 'resonance', 0.2);
engine.setParameter(rev, 'decay', 2.5);
engine.setParameter(rev, 'mix', 0.2);
```

### 2. ALWAYS layer for thickness
For pads: minimum 2-4 detuned oscillators + noise layer for breath.
For bass: saw/square + sub sine one octave below.
For drums: always add bus compression after mixing drum elements.

### 3. ALWAYS add movement
At least one LFO modulating filter cutoff, volume, or pan. Static = dead.

### 4. ALWAYS add space
Reverb on almost everything (mix 0.1-0.3). Delay for rhythmic depth. Chorus for width.

### 5. ALWAYS add texture
Subtle saturation/waveshaping. Noise layers mixed very low (-24dB) for "air."

### 6. Frequency stacking
Don't put competing elements in the same range. Bass below 250Hz. Pads 200-3000Hz. Hats above 6kHz.

## Common Mistakes — DO NOT DO THESE

```typescript
// ❌ Using raw Web Audio instead of Chord
const ctx = new AudioContext();
const osc = ctx.createOscillator();  // NO. Use Chord.

// ❌ Forgetting cleanup
useEffect(() => {
  const engine = new Chord();
  engine.start();
  // MISSING: return () => engine.stop();
}, []);

// ❌ Starting audio without user gesture
const engine = new Chord();
engine.start(); // FAILS in browsers — autoplay policy
// ✅ Correct:
button.onclick = () => engine.start();

// ❌ Single oscillator for a pad (sounds thin and raw)
const pad = engine.addNode('oscillator');
// ✅ Use multiple detuned oscillators + filter + reverb + LFO

// ❌ Reverb mix above 0.4 (washy, unprofessional)
engine.setParameter(rev, 'mix', 0.8);  // NO
engine.setParameter(rev, 'mix', 0.2);  // YES

// ❌ No limiter on master output (risks clipping)
// The Chord engine has a built-in master limiter — don't bypass it.

// ❌ Drums without humanization
// Always vary velocity ±15% and timing ±5ms for natural feel.
```
